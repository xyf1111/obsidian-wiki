# MySQL 与 Redis

> 来源：[xyf1111.github.io](https://xyf1111.github.io/)，整理自 CC 博客中的数据库系列文章。

## MySQL 学习

### 索引优化要点

- **复合索引**：遵循最佳左前缀原则，最常用的限制条件放最左边（如 `(area, age, salary)` 相当于 `(area,age,salary)`、`(area,age)`、`(area)` 三个索引）
- **索引失效场景**：`LIKE '%xxx%'`、列上运算、`NOT IN`、`<>`
- **避免 NULL**：含 NULL 值的列不会被包含在索引中
- **短索引**：对长字符串列指定前缀长度，节省空间和 I/O
- **排序**：`WHERE` 和 `ORDER BY` 用不同索引时只用一个，建议建复合索引

### MySQL 约束

- `NOT NULL`：字段值不能为空
- `UNIQUE`：值不能重复，但可为空
- `PRIMARY KEY`：唯一 + 非空
- `FOREIGN KEY`：表间关联，保证数据正确性

### MongoDB vs MySQL

| 维度 | MySQL | MongoDB |
|------|-------|---------|
| 类型 | 关系型 | 文档型 (NoSQL) |
| 存储 | 不同引擎不同方式 | 虚拟内存 + 持久化 |
| 查询 | SQL | MongoDB 查询语言 |
| 事务 | 支持 | 不支持 |
| 成熟度 | 高 | 较低 |
| 场景 | 结构化数据、强一致性 | 日志、内容管理、海量数据 |

## MySQL 面试核心问题

### 1. SQL 执行流程
连接器 → 查询缓存 → 分析器 → 优化器 → 执行器 → 存储引擎

### 2. 索引原理
B+ 树索引，主键索引叶子节点存整行数据，二级索引叶子节点存主键值（回表）

### 3. 事务隔离级别
- **读未提交**：脏读
- **读已提交**：不可重复读（InnoDB 通过 MVCC + ReadView 解决）
- **可重复读**（InnoDB 默认）：通过间隙锁解决部分幻读
- **串行化**：最高隔离级别，性能最差

### 4. 锁机制
- **行锁**：InnoDB，基于索引
- **表锁**：MyISAM
- **间隙锁**：锁定索引记录间的间隙，防止幻读
- **MDL 锁**：元数据锁，DDL 时使用

### 5. 高可用方案
- 主从复制（异步/半同步）
- MHA / Orchestrator
- 分库分表（ShardingSphere、Vitess）

## Redis 学习

### 数据结构与应用

| 类型 | 应用场景 |
|------|----------|
| String | 缓存、计数器、分布式锁 |
| Hash | 对象存储、用户信息 |
| List | 消息队列、时间线 |
| Set | 标签、共同好友 |
| ZSet | 排行榜、延迟队列 |

### 持久化

- **RDB**：快照，定时全量备份，恢复快但可能丢数据
- **AOF**：追加写命令，数据安全但文件大、恢复慢
- 混合持久化（Redis 5.0+）：RDB + 增量 AOF

### 高可用

- **主从复制**：读写分离，数据冗余
- **哨兵 (Sentinel)**：自动故障转移，监控 + 通知
- **Cluster**：数据分片，去中心化，16384 个槽位

## Redis 面试核心问题

### 缓存穿透/击穿/雪崩

| 问题 | 原因 | 解决 |
|------|------|------|
| 穿透 | 查询不存在的数据 | 布隆过滤器、缓存空值 |
| 击穿 | 热点 key 过期 | 互斥锁、永不过期+异步更新 |
| 雪崩 | 大量 key 同时过期 | 过期时间加随机值、多级缓存 |

### 分布式锁

- **SETNX + 过期时间**：`SET key value NX EX 30`
- **RedLock**（Redisson）：多节点投票，更可靠
- **注意事项**：
  - 锁需设置过期时间
  - 释放锁时验证 value（防止误删别人的锁）
  - 守护线程为将过期的锁续期

### Redis 单线程为什么快

- 纯内存操作
- 单线程避免上下文切换和锁竞争
- I/O 多路复用（epoll）
- 6.0+ 引入多线程处理网络 I/O，执行命令仍单线程
