# 项目换电脑部署指南

> 适用场景：换新电脑、重装系统后，恢复当前 Obsidian 知识库 + Quartz 静态站点项目。

## 项目结构

```
ObsidianVault/              ← 根目录（也是 Obsidian vault）
├── concepts/               ← 概念笔记
├── references/             ← 参考资料（博客笔记 93 篇等）
│   └── blog/               ← CC's Blog 文章（golang/mysql/redis/...）
├── skills/                 ← 技能文档
├── index.md                ← 站点索引
├── hot.md                  ← 近期活动
├── quartz/                 ← Quartz 静态站点生成器
│   ├── quartz.config.yaml  ← 站点配置
│   └── package.json        ← 依赖
├── .github/workflows/
│   └── deploy.yml          ← GitHub Actions 自动部署
└── .obsidian/              ← Obsidian 配置（已跟踪）
```

## 一键脚本

```bash
# === 1. 克隆仓库 ===
git clone https://github.com/xyf1111/obsidian-wiki.git ~/code/ObsidianVault
cd ~/code/ObsidianVault

# === 2. 创建 .env 文件（本地配置，不跟踪） ===
cat > .env << 'EOF'
OBSIDIAN_VAULT_PATH=~/code/ObsidianVault
OBSIDIAN_SOURCES_DIR=~/Documents,~/Downloads
CLAUDE_HISTORY_PATH=~/.claude
WIKI_TOKEN_WARN_THRESHOLD=100000
WIKI_STAGED_WRITES=false
EOF

# === 3. 用 Obsidian 打开 ===
# 打开 Obsidian → "Open folder as vault" → 选择此目录
# 或者命令行：
open obsidian://open?path=$HOME/code/ObsidianVault

# === 4. (可选) 本地预览 Quartz 站点 ===
cd quartz
npm ci                    # 安装依赖
npx quartz plugin install # 安装插件
npx quartz build -d ..    # 构建站点（内容来自上级目录）
npx quartz build -d .. --serve  # 启动本地预览 → http://localhost:8080
```

## 前置依赖

| 工具 | 作用 | 安装 |
|------|------|------|
| **Git** | 版本控制 | macOS 自带，或 `brew install git` |
| **Obsidian** | 笔记编辑 | [obsidian.md](https://obsidian.md/download) |
| **Node.js 22+** | Quartz 构建（仅本地预览需要） | `brew install node` 或 [nodejs.org](https://nodejs.org/) |

> Node.js 只在本地预览网站时需要。如果只是编辑笔记，不需要装。

## 日常使用

```bash
# 在 Obsidian 中写完笔记后：
cd ~/code/ObsidianVault
git add .
git commit -m "更新笔记内容"
git push

# GitHub Actions 会自动构建并部署到：
#   https://xyf1111.github.io/obsidian-wiki/
```

## 配置说明

### `.env`（本地，不跟踪）

包含本地路径配置，已在 `.gitignore` 中排除。换电脑后需重新创建，路径改为新电脑的实际情况。

### `.obsidian/`（已跟踪）

Obsidian 的编辑器配置已纳入版本控制，包括：
- `app.json` — 编辑器行为（live preview 等）
- `appearance.json` — 主题设置
- `core-plugins.json` — 核心插件开关

> 第三方插件和主题在 `.gitignore` 中排除（路径依赖），换电脑后需重新安装。

### `quartz/quartz.config.yaml`（已跟踪）

Quartz 站点配置：
- `pageTitle: "Wiki"`
- `locale: zh-CN`
- `baseUrl: xyf1111.github.io/obsidian-wiki`
- 搜索、图谱、反向链接等已开启

### `.github/workflows/deploy.yml`（已跟踪）

触发条件：`push` 到 `main` 分支
构建流程：`checkout → npm ci → quartz build → deploy to Pages`
